# README

## Описание

В рамках данного задания спроектирована объектная модель ML-сервиса аналитики криптовалютных активов.  
Модель отражает базовую бизнес-логику системы: работу с пользователями, ML-моделями, задачами на предсказание, результатами предсказаний, историей запросов и транзакциями.

## Реализованные сущности

В объектной модели представлены следующие базовые классы:

- `User` — пользователь системы;
- `Admin` — администратор, наследуется от пользователя;
- `Asset` — криптоактив, по которому выполняется анализ;
- `MLModel` — абстрактная ML-модель;
- `CryptoAnalysisModel` — конкретная реализация ML-модели;
- `PredictionTask` — задача на выполнение предсказания;
- `PredictionResult` — результат анализа;
- `PredictionHistory` — история ML-запросов пользователя;
- `Transaction` — абстрактная транзакция;
- `CreditTransaction` — пополнение баланса;
- `DebitTransaction` — списание баланса;
- `MLService` — сервис, координирующий выполнение задач и проведение транзакций.

## Принципы ООП

При проектировании использованы базовые принципы объектно-ориентированного программирования:

- **Инкапсуляция** — поля классов скрыты и доступны через свойства и методы;
- **Наследование** — реализовано расширение базовых классов (`Admin`, `CreditTransaction`, `DebitTransaction`, `CryptoAnalysisModel`);
- **Полиморфизм** — абстрактные методы `predict()` и `apply()` переопределяются в производных классах.

## Основная логика работы

Пользователь выбирает актив и ML-модель, после чего создаётся задача на предсказание.  
Система проверяет достаточность баланса, запускает выполнение задачи, получает результат анализа и сохраняет задачу в истории пользователя.  
После успешного выполнения с баланса пользователя списывается стоимость предсказания в виде транзакции.
